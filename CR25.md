# CR25 — Bullhorn MCP Bug Report Fixes

## Context

A stress test on Job #51227 (Interim CIO, Pinergy) surfaced four production-impacting bugs. All confirmed via live API testing before implementation.

1. **No reliable ID-based ClientCorporation lookup.** `search_entities(ClientCorporation, "id:9493")` returns `[]`; no `get_company` tool exists.
2. **`query_entities(ClientCorporation)` 400s** on the auto-appended `isDeleted` clause — ClientCorporation has no `isDeleted` column (uses `status` instead). Verified ClientCorporation IS queryable on `/query`; only the auto-clause breaks it.
3. **`get_notes_for_entity` is broken for every entity.** `_NOTE_DEFAULT_FIELDS` includes `clientCorporation(id,name)` which Bullhorn now rejects with `500 — Invalid field 'clientCorporation'` on the `/entity/{Entity}/{id}/notes` association endpoint. All entity types affected.
4. **`create_candidate` data quality.** Required-field env plumbing exists from CR19 but ships with no default list. `add_note` accepts any string as `action`. No `source` stamping for MCP-created records.

---

## Change set

### Fix 1 — `get_company` tool

- `server.py`: new `get_company(company_id, fields)` tool mirroring `get_job`/`get_candidate`.
- `descriptions.py`: add `"get_company": ["ClientCorporation"]` to `TOOL_ENTITY_MAP`.
- Tests: `TestGetCompany` class.

### Fix 2 — Client-level `isDeleted` denylist

- `client.py`: `_ENTITIES_WITHOUT_ISDELETED = {"ClientCorporation", "UserMessage"}`. `search()` and `query()` skip the auto-append when entity is in the set, regardless of `exclude_deleted` value.
- `server.py`: remove now-redundant `exclude_deleted=False` from `search_emails` (CR24 workaround).
- Tests: denylist tests in `test_client.py`.

### Fix 3 — Restore `get_notes_for_entity`

- `server.py`: drop `clientCorporation(id,name)` from `_NOTE_DEFAULT_FIELDS`. The comment on `_NOTE_SEARCH_DEFAULT_FIELDS` is updated to reflect both fields now share the same exclusion.
- Tests: add `ClientContact` and `JobOrder` note-retrieval tests so this can't silently regress.

### Fix 4a — Ship default required-fields list

- `.env.example`: uncomment `BULLHORN_CANDIDATE_REQUIRED` with the full recommended list.
- `.env`: apply the same list.
- `server.py`: docstring nudge in `create_candidate` / `create_candidate_from_cv`.

### Fix 4b — Validate `add_note` action

- `server.py`: `_valid_note_actions: set[str] | None = None` module-level cache; `_load_valid_note_actions(metadata)` helper reads from cached Note metadata. `add_note` validates against this set; returns structured error with valid list on mismatch. Gracefully skips validation if picklist load fails.

### Fix 4c — Auto-stamp `source` on MCP-created candidates

- `candidate_config.py`: `get_mcp_source() -> str` reading `BULLHORN_MCP_SOURCE` env var, defaulting to `"Claude"`.
- `server.py`: stamp `source = get_mcp_source()` in `create_candidate` and `create_candidate_from_cv` when source is not already set. User-supplied value wins.
- `.env.example`: new `BULLHORN_MCP_SOURCE=Claude` entry.
- Tests: source auto-stamping coverage.

### Fix 4d — Deferred

Skill design for guided candidate creation deferred to separate task.
