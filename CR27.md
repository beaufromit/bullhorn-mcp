# CR27 — `POST /upload-cv` HTTP endpoint for automated CV uploads

## Context

The MCP server runs in HTTP mode behind Entra OIDC. When an automated workflow (a
container running Claude, an email-to-Bullhorn pipeline, etc.) needs to upload a
CV file, base64-encoding the bytes and passing them through the MCP tool layer is
slow and renders the entire base64 string in the conversation context.

This CR adds a dedicated `POST /upload-cv` HTTP route on the same ASGI app as the
MCP transport. It accepts the file as raw multipart bytes (no base64 in the
conversation layer), authenticates via a pre-shared `X-Upload-Secret` header
(no Entra token needed), and proxies the upload through the existing
authenticated Bullhorn session.

## Verification notes from exploration

| Claim in original prompt | Verified? | Notes |
|---|---|---|
| `mcp.custom_route()` exists | ✅ | Provided by `TransportMixin` in installed FastMCP 2.x (`fastmcp/server/mixins/transport.py`). Decorator signature: `custom_route(path, methods, name=None, include_in_schema=True)`. Handler must be `async (Request) -> Response`. |
| "Bypasses Entra auth" | ⚠️ Wording inaccurate | FastMCP only wraps the MCP transport endpoint with `RequireAuthMiddleware`. Custom routes are **anonymous by default** — there is nothing to bypass. We are adding `X-Upload-Secret` as the sole auth on a new public route. |
| Reuse `create_candidate_from_cv` | ✅ | Tests already call the decorated tool function directly as `server.create_candidate_from_cv(...)`. The `@mcp.tool()` decorator preserves callability. |
| Reuse `get_client()` | ✅ | Module-level singleton, safe to reuse. |
| `client.attach_file(...)` signature | ✅ | Matches the prompt — but the prompt does not specify how to derive `content_type`. Use `client._guess_content_type(format)`. |
| Doesn't break stdio mode | ✅ | `custom_route` registration happens at import time but the route is never served when `MCP_TRANSPORT=stdio`. |
| `mcp.tool()` functions are sync | ⚠️ Implication | The custom-route handler must be `async`, but `create_candidate_from_cv` and `client.attach_file` are sync (blocking httpx). Wrap calls in `asyncio.to_thread(...)` to avoid blocking the event loop. |

## Design

### Route

`POST /upload-cv` registered via `@mcp.custom_route("/upload-cv", methods=["POST"])`
in `server.py`. Handler signature:

```python
async def _upload_cv_handler(request: Request) -> Response
```

The handler is also exported at module level so tests can call it directly
(matching the dominant "call the tool function directly" pattern in
`test_server.py`).

### Request — `multipart/form-data`

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `file` | bytes | yes | — | The CV file bytes |
| `filename` | str | yes | — | Original filename |
| `format` | str | no | `"pdf"` | One of `pdf`/`doc`/`docx`/`html`/`text` |
| `candidate_id` | int | no | — | If set, attach to this Candidate; else parse-and-create |
| `force` | str | no | `"false"` | `"true"`/`"false"` — bypass duplicate check on create path |

`force` is parsed as `value.lower() == "true"` (form fields are always strings).

### Authentication

Single header: `X-Upload-Secret: <value>` matched against the `UPLOAD_SECRET`
environment variable. Logic order matters:

1. `UPLOAD_SECRET` env var not set → `400` with `{"error": "upload_secret_not_configured"}`. Endpoint refuses to operate at all without a secret.
2. `X-Upload-Secret` header missing or mismatched → `401` with `{"error": "unauthorized"}`. Use `hmac.compare_digest` for constant-time comparison.

Auth failures logged at `WARNING` (no secret value in logs).

### Branching

**`candidate_id` not provided (create path):**

1. Base64-encode the file bytes.
2. Call `server.create_candidate_from_cv(file_b64=<b64>, filename=<name>, format=<fmt>, force=<bool>)` via `asyncio.to_thread`.
3. The tool returns a JSON string — return as `Response(body=<that string>, media_type="application/json", status_code=200)`.

**`candidate_id` provided (attach path):**

1. `client = get_client()`.
2. `content_mime = client._guess_content_type(format)`.
3. Call `client.attach_file("Candidate", candidate_id, file_bytes, filename, content_mime, file_type="CV")` via `asyncio.to_thread`.
4. Return the dict as JSON with status `200`.

### Error handling

| Scenario | Status | Body |
|---|---|---|
| Missing `file` or `filename` | 400 | `{"error": "missing_field", "message": "..."}` |
| Bad `candidate_id` (not int) | 400 | `{"error": "invalid_candidate_id"}` |
| Any `BullhornAPIError` | 500 | `{"error": "bullhorn_error", "message": str(exc)}` |
| Any other exception | 500 | `{"error": "internal_error", "message": str(exc)}` |

All responses are JSON (`starlette.responses.JSONResponse`). Never HTML.

### Logging

- INFO on every attempt: `"upload-cv attempt filename=%s candidate_id=%s"` and a follow-up `"upload-cv success ..."` / `"upload-cv error ..."`.
- WARNING on auth failure: `"upload-cv auth failure from %s"` (client host only, no secret value).

## Files to modify

### `src/bullhorn_mcp/server.py`

Add near the end of the file, after existing tool definitions:

```python
import hmac
from starlette.requests import Request
from starlette.responses import JSONResponse, Response

async def _upload_cv_handler(request: Request) -> Response:
    upload_secret = os.environ.get("UPLOAD_SECRET")
    if not upload_secret:
        return JSONResponse({"error": "upload_secret_not_configured"}, status_code=400)

    provided = request.headers.get("X-Upload-Secret", "")
    if not provided or not hmac.compare_digest(provided, upload_secret):
        _logger.warning("upload-cv auth failure from %s", request.client.host if request.client else "?")
        return JSONResponse({"error": "unauthorized"}, status_code=401)

    form = await request.form()
    upload = form.get("file")
    filename = form.get("filename")
    fmt = (form.get("format") or "pdf")
    candidate_id_raw = form.get("candidate_id")
    force_raw = (form.get("force") or "false")

    if upload is None or not filename:
        return JSONResponse({"error": "missing_field", "message": "file and filename are required"}, status_code=400)

    file_bytes = await upload.read() if hasattr(upload, "read") else bytes(upload)
    force = str(force_raw).lower() == "true"

    candidate_id: int | None = None
    if candidate_id_raw:
        try:
            candidate_id = int(candidate_id_raw)
        except (TypeError, ValueError):
            return JSONResponse({"error": "invalid_candidate_id"}, status_code=400)

    _logger.info("upload-cv attempt filename=%s candidate_id=%s", filename, candidate_id)

    try:
        if candidate_id is not None:
            client = get_client()
            content_mime = client._guess_content_type(fmt)
            result = await asyncio.to_thread(
                client.attach_file, "Candidate", candidate_id,
                file_bytes, filename, content_mime, None, "CV",
            )
            _logger.info("upload-cv success attach candidate_id=%s", candidate_id)
            return JSONResponse(result, status_code=200)
        else:
            file_b64 = base64.b64encode(file_bytes).decode()
            result_json = await asyncio.to_thread(
                create_candidate_from_cv,
                file_b64=file_b64, filename=filename, format=fmt, force=force,
            )
            _logger.info("upload-cv success create filename=%s", filename)
            return Response(content=result_json, media_type="application/json", status_code=200)
    except BullhornAPIError as exc:
        _logger.exception("upload-cv bullhorn error filename=%s", filename)
        return JSONResponse({"error": "bullhorn_error", "message": str(exc)}, status_code=500)
    except Exception as exc:
        _logger.exception("upload-cv internal error filename=%s", filename)
        return JSONResponse({"error": "internal_error", "message": str(exc)}, status_code=500)


mcp.custom_route("/upload-cv", methods=["POST"])(_upload_cv_handler)
```

Notes:
- `mcp.custom_route(...)` is called as a function (not `@`) so the handler stays
  importable for tests. The decorator returns the same function reference, so
  this is equivalent.
- `attach_file` is called positionally because its kwargs are `external_id` then
  `file_type`. Passing `None, "CV"` matches the signature in `client.py:148`.

Actually the simplest reliable approach is to call the handler with a minimal
fake request object that exposes only the attributes the handler reads
(`headers`, `client`, `form()`). A small helper:

```python
class _FakeRequest:
    def __init__(self, form_fields, headers):
        self.headers = headers
        self.client = None
        self._form = form_fields
    async def form(self):
        return self._form
```

For the `file` field, wrap bytes in a small stub:

```python
class _FakeUpload:
    def __init__(self, data: bytes):
        self._data = data
    async def read(self):
        return self._data
```
