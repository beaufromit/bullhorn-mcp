# CR29 — `get_contact` tool

## Problem

Every other primary entity has a dedicated `get_` tool (`get_job`, `get_candidate`, `get_company`) but ClientContact does not. Consultants who want to inspect a single contact by ID must fall back to `query_entities`, which requires knowing field names and Bullhorn SQL syntax. This is a parity gap.

## Solution

Add a `get_contact(contact_id, fields=None)` tool to `server.py` mirroring the exact pattern of `get_job`, `get_candidate`, and `get_company`. Add `"get_contact": ["ClientContact"]` to `TOOL_ENTITY_MAP` in `descriptions.py` so it picks up the startup field-reference section.

## Assumptions

1. Default fields should cover the fields a consultant would want at a glance: `id,firstName,lastName,name,email,phone,occupation,status,clientCorporation,owner,dateAdded`.
2. No pagination — this is a single-record lookup by ID, same as all other `get_` tools.
3. No new client method needed — `client.get()` already handles this.
4. `ClientContact` is already in `SUPPORTED_ENTITIES` in `descriptions.py` so no change needed there.

## Changes

### `src/bullhorn_mcp/server.py`

Add after `get_company` (around line 562):

```python
@mcp.tool()
def get_contact(contact_id: int, fields: str | None = None) -> str:
    """Get details for a specific client contact by ID.

    Args:
        contact_id: The ClientContact ID
        fields: Comma-separated fields to return
                (default: id,firstName,lastName,name,email,phone,occupation,
                 status,clientCorporation,owner,dateAdded)

    Returns:
        JSON object with contact details.

    Note on notes: use get_notes_for_entity("ClientContact", contact_id) for contact notes.
    """
    try:
        client = get_client()
        result = client.get(entity="ClientContact", entity_id=contact_id, fields=fields)
        return format_response(result)

    except (AuthenticationError, BullhornAPIError) as e:
        return f"ERROR: {e}"
```

### `src/bullhorn_mcp/descriptions.py`

In `TOOL_ENTITY_MAP`, add:

```python
"get_contact": ["ClientContact"],
```

alongside the other `get_` entries.
